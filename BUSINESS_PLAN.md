# Краткий бизнес-план ML Inference

Ниже — **УТП**, **модель доходов**, **риски**, а также связь **биллинга** и **промокодов** с продуктом.

---

## 1. Уникальное торговое предложение (УТП)

- **Прозрачное ценообразование в кредитах**: пользователь платит только за **успешно завершённые** предсказания; стоимость одного запроса задаётся **настраиваемой ценой в кредитах**.
- **Асинхронная обработка**: HTTP API не выполняет тяжёлый инференс в запросе; очередь **Celery** и горизонтальное добавление воркеров дают масштабируемость.
- **Гибкий биллинг**: журнал транзакций, идемпотентное списание, промокоды на **фиксированные кредиты** и на **бонус к следующему пополнению** (процент).
- **Наблюдаемость**: метрики Prometheus, дашборды Grafana — контроль нагрузки и качества сервиса.

---

## 2. Модель доходов и расходов

| Поток | Описание |
|-------|----------|
| **Пополнение** | В продукте — оплата через **платёжный шлюз**, webhook начисляет кредиты с **идемпотентностью** по внешнему ID платежа. В демо-сборке без эквайринга — **mock-пополнение** по секрету (имитация платежа). |
| **Расход** | Списание кредитов **только после успешного** выполнения задачи предсказания; **неуспешные** задачи (ошибка модели, валидации и т.д.) **не списывают** баланс. |
| **Промо** | Коды на разовое начисление кредитов; коды на **скидку/бонус к следующему** пополнению — стимулируют повторные платежи без демпинга маржи на текущем пополнении. |

---

## 3. Промокоды (продуктовая логика)

- У кода есть **срок действия** и **лимит числа активаций** (глобально по коду).
- **Один пользователь — одна активация на код** (защита от повторного использования на уровне данных и интерфейса).
- Пример кода для онбординга пользователей: **`WELCOME`** (разовое начисление кредитов при первом использовании продукта).

---

## 4. Unit economics (упрощённо)

- Условно **1 кредит = 0.10 USD** (пример), **C** — себестоимость одного предсказания (инфраструктура, хранение моделей, поддержка).
- Маржа с успешного запроса: `цена_в_кредитах × курс_кредита − C`.
- **Рычаги**: размер пакетов кредитов, промо только на **следующее** пополнение (процентный тип), мониторинг SLA воркеров (Grafana, алерты в перспективе).

---

## 5. Риски и меры

| Риск | Мера в продукте |
|------|-----------------|
| **Двойное списание** | Идемпотентные ключи в журнале транзакций; повторная обработка задачи не дублирует дебет. |
| **Сбой между инференсом и списанием** | Списание оформлено в **одной транзакции БД** с успешным завершением задачи; при откате кредиты не списываются. |
| **Мошенничество с платежами** | В продакшене: проверка подписи webhook, идемпотентность по внешнему идентификатору платежа, секреты не хранятся в коде. |
| **Повторная активация промокода** | Один пользователь не активирует один и тот же код дважды; при повторной попытке — отказ с сигналом конфликта. |
