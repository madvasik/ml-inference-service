# E2E Тесты реального использования

Этот набор скриптов предназначен для комплексного тестирования реальных сценариев использования ML Inference Service. Тесты проверяют:

- ✅ Отсутствие падений при действиях пользователей
- ✅ Согласованность данных между Backend, Prometheus и Grafana
- ✅ Корректность работы всех систем в реальных условиях
- ✅ Стабильность под нагрузкой
- ✅ Обработку ошибок и граничных случаев

## Быстрый старт

### Запуск всех E2E тестов

```bash
BASE_URL=http://localhost:8000 \
PROMETHEUS_URL=http://localhost:9090 \
GRAFANA_URL=http://localhost:3000 \
bash scripts/testing/run_all_e2e_tests.sh
```

### Запуск отдельных тестов

#### 1. Тест стабильности (отсутствие падений)

```bash
BASE_URL=http://localhost:8000 python3 scripts/testing/test_no_crashes.py
```

**Что проверяет:**
- Быстрые операции подряд
- Одновременная работа нескольких пользователей
- Восстановление после ошибок
- Длительная сессия (30+ секунд)

#### 2. Тест реальных пользовательских сценариев

```bash
BASE_URL=http://localhost:8000 \
PROMETHEUS_URL=http://localhost:9090 \
GRAFANA_URL=http://localhost:3000 \
python3 scripts/testing/test_real_world_scenarios.py
```

**Что проверяет:**
- **Сценарий 1**: Новый пользователь - полный цикл (регистрация → пополнение → загрузка модели → предсказания)
- **Сценарий 2**: Несколько пользователей работают одновременно
- **Сценарий 3**: Согласованность метрик Backend ↔ Prometheus ↔ Grafana
- **Сценарий 4**: Обработка ошибок и граничных случаев
- **Сценарий 5**: Согласованность данных API ↔ БД
- **Сценарий 6**: Стабильность под нагрузкой
- **Сценарий 7**: Админские операции (просмотр всех пользователей, предсказаний, транзакций с фильтрацией)
- **Сценарий 8**: Управление моделями (загрузка, просмотр, удаление)
- **Сценарий 9**: Пагинация и фильтрация данных
- **Сценарий 10**: Проверка безопасности (доступ к чужим данным)
- **Сценарий 11**: История транзакций и предсказаний
- **Сценарий 12**: Rate limiting в реальных условиях
- **Сценарий 13**: Большой объем данных (множество моделей и предсказаний)
- **Сценарий 14**: Валидация входных данных
- **Сценарий 15**: Проверка метрик в реальном времени

#### 3. Проверка согласованности с Grafana

```bash
BASE_URL=http://localhost:8000 \
PROMETHEUS_URL=http://localhost:9090 \
GRAFANA_URL=http://localhost:3000 \
GRAFANA_USER=admin \
GRAFANA_PASSWORD=admin \
python3 scripts/testing/test_grafana_consistency.py
```

**Что проверяет:**
- Подключение Prometheus datasource в Grafana
- Трехстороннее сравнение метрик (Backend ↔ Prometheus ↔ Grafana)
- Проверка данных на дашбордах Grafana
- Выявление несоответствий между системами

## Требования

- Python 3.11+
- Установленная библиотека `requests`: `pip install requests`
- Запущенные сервисы:
  - Backend (http://localhost:8000)
  - Prometheus (http://localhost:9090) - опционально
  - Grafana (http://localhost:3000) - опционально
  - Celery worker - должен быть запущен для обработки предсказаний

## Переменные окружения

- `BASE_URL` - URL API сервиса (по умолчанию: `http://localhost:8000`)
- `PROMETHEUS_URL` - URL Prometheus (по умолчанию: `http://localhost:9090`)
- `GRAFANA_URL` - URL Grafana (по умолчанию: `http://localhost:3000`)
- `GRAFANA_USER` - Пользователь Grafana (по умолчанию: `admin`)
- `GRAFANA_PASSWORD` - Пароль Grafana (по умолчанию: `admin`)

## Что проверяют тесты

### test_no_crashes.py

1. **Быстрые операции подряд** - выполняет множество операций быстро, проверяя отсутствие падений
2. **Одновременные пользователи** - несколько пользователей работают параллельно
3. **Восстановление после ошибок** - система должна корректно обрабатывать ошибки и продолжать работать
4. **Длительная сессия** - операции в течение 30+ секунд

### test_real_world_scenarios.py

1. **Сценарий 1: Новый пользователь**
   - Регистрация
   - Пополнение баланса
   - Загрузка ML модели
   - Создание предсказаний
   - Проверка баланса и транзакций

2. **Сценарий 2: Параллельная работа**
   - Несколько пользователей одновременно
   - Параллельное создание предсказаний
   - Проверка обработки всех запросов

3. **Сценарий 3: Согласованность метрик**
   - Сравнение метрик между Backend и Prometheus
   - Проверка согласованности данных

4. **Сценарий 4: Обработка ошибок**
   - Несуществующая модель
   - Недостаточный баланс
   - Невалидный токен
   - Невалидные данные

5. **Сценарий 5: Согласованность данных**
   - Проверка ссылок между таблицами
   - Согласованность балансов с транзакциями
   - Целостность данных

6. **Сценарий 6: Стабильность под нагрузкой**
   - Множество запросов подряд
   - Проверка отсутствия падений

7. **Сценарий 7: Админские операции**
   - Просмотр всех пользователей
   - Просмотр всех предсказаний
   - Просмотр всех транзакций
   - Фильтрация по user_id и model_id

8. **Сценарий 8: Управление моделями**
   - Загрузка нескольких моделей
   - Просмотр списка моделей
   - Получение информации о модели
   - Удаление модели

9. **Сценарий 9: Пагинация и фильтрация**
   - Пагинация предсказаний
   - Пагинация транзакций
   - Фильтрация по model_id
   - Фильтрация по user_id
   - Проверка отсутствия дубликатов

10. **Сценарий 10: Проверка безопасности**
    - Доступ к чужим моделям
    - Доступ к чужим предсказаниям
    - Удаление чужих моделей
    - Изоляция данных между пользователями

11. **Сценарий 11: История и аудит**
    - История транзакций
    - История предсказаний
    - Сортировка по дате
    - Детали конкретных операций

12. **Сценарий 12: Rate Limiting**
    - Множество запросов подряд
    - Проверка срабатывания rate limit
    - Корректная обработка блокировок

13. **Сценарий 13: Большой объем данных**
    - Загрузка множества моделей
    - Создание множества предсказаний
    - Проверка доступности всех данных

14. **Сценарий 14: Валидация входных данных**
    - Отрицательные суммы
    - Нулевые значения
    - Невалидные типы данных
    - Невалидный email
    - Отсутствие обязательных полей

15. **Сценарий 15: Метрики в реальном времени**
    - Изменение метрик при действиях пользователя
    - Сравнение метрик до и после операций
    - Доступность метрик Prometheus

### test_grafana_consistency.py

1. **Проверка Prometheus datasource**
   - Наличие datasource в Grafana
   - Корректность настроек
   - Доступность через Grafana

2. **Трехстороннее сравнение метрик**
   - Backend ↔ Prometheus
   - Prometheus ↔ Grafana
   - Backend ↔ Grafana

3. **Проверка дашбордов**
   - Наличие дашбордов ML Service
   - Доступность данных

1. Логи Backend: `docker-compose logs backend`
2. Логи Celery: `docker-compose logs celery`
3. Prometheus метрики: http://localhost:9090
4. Grafana дашборды: http://localhost:3000

## Рекомендации

1. **Запускайте тесты после инициализации данных:**
   ```bash
   bash scripts/setup/init_data_api.sh
   ```

2. **Убедитесь, что Celery worker запущен:**
   ```bash
   docker-compose ps celery
   ```

3. **Проверьте доступность всех сервисов перед запуском тестов**

4. **Для production-подобного тестирования используйте реальные данные**

## Интеграция в CI/CD

Эти тесты можно интегрировать в CI/CD pipeline:

```yaml
# Пример для GitHub Actions
- name: Run E2E Tests
  run: |
    docker-compose up -d
    sleep 30  # Ожидание запуска сервисов
    bash scripts/testing/run_all_e2e_tests.sh
```
